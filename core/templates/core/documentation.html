Platform Documentation

Comprehensive guide to the ShieldML Dual-Engine SMS Spam Prediction Framework.

🌍 1. Project Overview

ShieldML is an advanced messaging security framework constructed to intercept, classify, and isolate mobile threat vectors—specifically targeting phishing, OTP scams, and lottery fraud.

Relying on a dual-pipeline architecture, our platform offers both a lightning-fast heuristic engine (XGBoost) utilizing TF-IDF vectors, and a deep-context natural language engine (DistilBERT) utilizing transformer models to catch highly sophisticated attacks.

🏢 3. Real-World Use Cases

Our infrastructure is designed for scalability and adaptation across multi-tier environments:

  • Enterprise SMS Gateways: Clean outgoing/incoming messaging traffic from API gateways (e.g., Twilio, Plivo) automatically before it hits the consumer.
  • Personal Client Dashboard: Individual users log in and immediately see their dedicated Shield history. They can manually parse confusing or suspicious messages sent by unknown numbers to get an instant safety report before clicking any embedded links.
  • Crowdsourced Intelligence: The Public Scanner aggregates global threat data from non-authenticated users. This feeds into our broader Machine Learning retraining pipeline to continuously adapt to zero-day spam campaigns, sharing threat intelligence safely.

🛠️ 4. Quick-Start Guide for Users

Using the system is designed to be completely frictionless:

  1. Login or Register: Secure an account to keep your prediction history completely private and sandboxed.
  2. Navigate to the Dashboard: View your historical trends, engine breakdown, and aggregate metrics.
  3. Access the Predictor: Paste the text of an SMS or URL.
  4. Select your Engine: Pick XGBoost for instant checking against known patterns, or DistilBERT for deep contextual reading (useful for subtle social engineering).
  5. Analyze the Log: View your database records securely to trace the status and human-correct our engines if a false positive occurs.
